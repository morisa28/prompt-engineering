# Customer Service QA

## 1. Purpose
用于为客服对话质检、工单分析、通话转写分析、服务质量评分、违规话术识别、客户情绪分析、客诉归因、坐席培训反馈和质检规则设计生成 prompt。本分支要求所有评价绑定对话证据，并保护隐私。

## 2. Trigger Conditions
- 用户提到客服质检、工单分析、通话转写、服务评分、违规话术、坐席培训。
- 用户要求分析客户情绪、客诉归因、坐席表现或改进建议。
- 用户要求设计质检规则、评分表或批量质检报告。

## 3. Required Inputs
- {{conversation_transcript}} 对话、工单或通话转写。
- {{business_context}} 业务场景。
- {{service_channel}} 服务渠道。
- {{quality_dimensions}} 质检维度。
- {{scoring_rubric}} 评分标准。
- {{compliance_rules}} 合规或违规规则。
- {{customer_sentiment_focus}} 客户情绪关注点。
- {{privacy_requirements}} 隐私脱敏要求。
- {{output_audience}} 输出读者。

缺失信息处理：
- 评分标准缺失时先要求生成可确认的 rubric，不直接给员工结论。
- 对话证据不足时标注无法判断，不用主观印象补全。
- 涉及合规条款时组合 `legal-policy-review`，并标注信息性分析边界。

## 4. Prompt Construction Rules
- 必须先明确质检维度、评分等级、扣分规则和合规红线。
- 必须要求基于对话证据评分，每个扣分或表扬点都引用片段。
- 必须区分事实、推断、情绪判断和改进建议。
- 必须要求隐私脱敏，隐藏姓名、电话、地址、订单号、身份证和账号等信息。
- 必须避免人身评价，只评价对话行为、流程遵守和可训练技能。
- 必须输出结构化评分表、问题归因、改进建议和培训话术。
- 批量对话必须统计问题类型、频次、严重程度和样例片段。

## 5. Hard Constraints
- 禁止凭感觉评价坐席或客户。
- 禁止泄露客户隐私或内部敏感信息。
- 禁止把客户情绪推断写成确定事实。
- 禁止只有批评没有可执行改进建议。
- 必须引用原始对话证据并标注不确定内容。

## 6. Output Format
最终 prompt 应要求目标模型输出：
- 对话概览和脱敏说明。
- 评分表。
- 证据片段。
- 违规或风险项。
- 客户情绪和客诉归因。
- 坐席改进建议。
- 培训话术或下一步行动。

## 7. Quality Checklist
- [ ] 业务场景、渠道和读者明确。
- [ ] 质检维度和评分标准明确。
- [ ] 每个评分点要求引用对话证据。
- [ ] 事实、推断和建议分开。
- [ ] 隐私脱敏规则明确。
- [ ] 避免人身评价，只评价行为。
- [ ] 输出结构化评分表和改进建议。
- [ ] 合规风险有边界和证据。

## 8. Common Mistakes
| Common Mistake | Risk | Repair |
| --- | --- | --- |
| 没有评分标准 | 评价不可复核 | 先定义维度、权重和扣分规则 |
| 凭感觉评价 | 对员工不公平 | 每项结论绑定对话片段 |
| 没有引用证据 | 无法申诉或训练 | 输出原句片段和时间/轮次 |
| 泄露隐私 | 合规风险 | 明确脱敏字段和输出边界 |
| 情绪推断写成事实 | 误判客户 | 标注“推断”并给证据 |
| 只批评不建议 | 无改进路径 | 给替代话术和训练建议 |

## 9. Reusable Template
```text
你是 {{target_ai_tool}}，请进行客服质检。

对话材料：{{conversation_transcript}}
业务场景：{{business_context}}
服务渠道：{{service_channel}}
质检维度：{{quality_dimensions}}
评分标准：{{scoring_rubric}}
合规规则：{{compliance_rules}}
情绪关注：{{customer_sentiment_focus}}
隐私要求：{{privacy_requirements}}
输出读者：{{output_audience}}

执行要求：
1. 先按隐私要求脱敏，再分析。
2. 按质检维度评分，每个评分点必须引用对话证据。
3. 区分事实、推断、建议和无法判断内容。
4. 识别违规话术、流程遗漏、客户情绪和客诉归因。
5. 给出可执行改进建议和替代话术，避免人身评价。

输出格式：
脱敏说明、评分表、证据片段、风险项、情绪/归因、改进建议、培训话术、待确认问题。
```

## 10. Example
用户原始需求：

```text
帮我分析客服对话质量并给坐席改进建议。
```

高质量 prompt：

```text
请质检以下电商售后客服对话 `[conversation_transcript]`。渠道是在线聊天，读者是客服主管。先脱敏姓名、手机号、订单号和地址；再按响应速度、问题理解、流程合规、情绪安抚、解决方案、结束确认六个维度评分。每个扣分或表扬点必须引用对话轮次和原句片段。请区分事实、客户情绪推断和改进建议；禁止评价坐席人格。输出评分表、证据、违规风险、客诉归因、替代话术和培训建议。
```
