# Roleplay Simulation

## 1. Purpose
用于设定角色、场景、目标、难度和反馈方式，支持迭代训练。 本分支只处理该场景专属 prompt 构造；跨场景原则使用 `../../common-principles.md`。

## 2. Trigger Conditions
- 用户要求面试/谈判/客服模拟
- 用户要求销售话术演练
- 用户要求英语口语练习

## 3. Required Inputs
- {{role_a}} 用户角色
- {{role_b}} 对方角色
- {{scenario}} 场景
- {{goal}} 训练目标
- {{difficulty}} 难度

缺失信息处理：
- 难度未知时从中等开始并可升级
- 反馈偏好未知时每轮后给简短反馈
- 涉及安全、金钱、医疗、法律、生产数据或大范围改动时，缺失的关键边界必须标注为阻塞问题。
- 非阻塞缺失项使用 `[待补充: field]`，并在最终 prompt 中要求目标模型标注假设。

## 4. Prompt Construction Rules
- 明确角色和场景
- 控制难度
- 每轮只推进一个交互
- 提供反馈和可替代表达
- 支持多轮迭代训练
- 需要适配 Codex、Codex CLI、Claude Code、Gemini CLI 或 ChatGPT 时，必须加入目标工具的工作方式、权限边界和最终报告格式。

## 5. Hard Constraints
- 禁止一次性给完整剧本代替互动
- 禁止脱离目标角色
- 禁止反馈只有鼓励没有可改动作
- 必须包含可检查的验收标准。
- 不确定时必须标注假设或提出阻塞问题。

## 6. Output Format
最终 prompt 应要求目标模型输出：
- 角色设定
- 规则
- 第一轮对话
- 反馈维度
- 下一轮提示

## 7. Quality Checklist
- [ ] 角色边界明确
- [ ] 目标可训练
- [ ] 难度可调
- [ ] 反馈可执行
- [ ] prompt 包含目标、输入、步骤、约束、输出格式、验收标准和自检。
- [ ] 目标工具的执行环境和限制已写明。

## 8. Common Mistakes
| Common Mistake | Risk | Repair |
| --- | --- | --- |
| 直接给答案稿 | 无法训练 | 多轮互动 |
| 反馈空泛 | 不能进步 | 按内容/语气/策略给建议 |
| 难度不稳 | 训练失衡 | 设定难度阶梯 |

## 9. Reusable Template
```text
你是 {{target_ai_tool}}，请处理以下 Roleplay Simulation 任务。

任务目标：
{{task_goal}}

输入材料：
- {{role_a}}
- {{role_b}}
- {{scenario}}
- {{goal}}
- {{difficulty}}
- {{feedback_style}}
- {{input_materials}}

执行要求：
1. 明确角色和场景
2. 控制难度
3. 每轮只推进一个交互
4. 提供反馈和可替代表达
5. 支持多轮迭代训练

硬性约束：
- 禁止一次性给完整剧本代替互动
- 禁止脱离目标角色
- 禁止反馈只有鼓励没有可改动作
- 必须按 {{output_format}} 输出。
- 必须满足 {{acceptance_criteria}}。

验证与自检：
- 完成后检查目标、输入、约束、输出格式和验收标准是否全部满足。
- 对缺失或不确定的信息标注“假设”或“待补充”。
- 如果无法完成，说明阻塞原因和下一步需要的输入。
```

## 10. Example
用户原始需求：

```text
帮我写一个面试模拟 prompt。
```

高质量 prompt：

```text
请扮演一名高级前端面试官，和我进行 React 岗位模拟面试。难度中高，目标是训练项目表达和系统设计。一次只问一个问题；我回答后，你按结构、技术深度、表达清晰度和可改进点给反馈，再追问下一题。不要直接给标准答案，除非我要求复盘。
```
