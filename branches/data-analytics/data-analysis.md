# Data Analysis

## 1. Purpose

用于设计可复现的数据分析 prompt，指导 agent 读取数据、理解字段、清洗、定义指标、分析问题、生成图表、输出结论、限制和下一步建议。

不用于编造数据结论、跳过数据质量检查、把相关性写成因果或处理没有数据来源的泛化观点。

## 2. Trigger Conditions

明确命中：
- 用户要求分析数据集、趋势、异常、指标、实验或业务数据。
- 用户要求输出数据报告、洞察、图表建议或可复现分析代码。
- 用户提供 CSV、数据库、日志、表格、实验结果或数据字段。

可能命中：
- 表格分析可使用 `spreadsheet-analysis` 作为主分支或辅助。
- 可视化报告可加入 `visualization-dashboard` 和 `report-writing`。

不应命中：
- 用户只要求普通报告写作，使用 `report-writing`。
- 用户只要求设计仪表盘，使用 `visualization-dashboard`。
- 数据涉及投资建议时，主分支应使用 `finance-investment-analysis` 并加安全边界。

相似分支区别：
- `spreadsheet-analysis` 更适合 Excel/CSV 操作和透视。
- `visualization-dashboard` 更适合图表和仪表盘交付。
- `report-writing` 负责最终文稿结构。

## 3. Required Inputs

必需输入：
- `{{data_source}}` 数据来源和路径。
- `{{analysis_questions}}` 分析问题。
- `{{field_definitions}}` 字段含义。
- `{{metric_definitions}}` 指标口径。
- `{{time_range}}` 时间范围。
- `{{deliverable}}` 交付物。

可选输入：
- `{{cleaning_rules}}` 清洗规则。
- `{{method_constraints}}` 方法限制。
- `{{visualization_requirements}}` 图表要求。
- `{{audience}}` 读者。
- `{{privacy_rules}}` 隐私和脱敏要求。

缺失时处理：
- 字段含义未知：先建立数据字典和待确认项。
- 指标口径未知：提出候选口径并标注待确认，不直接下结论。
- 数据源缺失：阻塞，不能生成具体分析结论。
- 涉及个人数据：要求脱敏、最小化展示和限制导出。

## 4. Prompt Construction Rules

- 目标定义：把分析问题写成可回答的问题或假设。
- 数据理解：读取 schema、样例行、字段类型、时间范围和样本量。
- 质量检查：缺失、重复、异常、类型错误、极值、口径冲突、采样偏差。
- 清洗规则：所有过滤、转换、聚合、去重必须记录。
- 指标定义：明确分子、分母、时间窗口、分组维度和排除条件。
- 分析步骤：输出可复现代码、查询或步骤。
- 可视化：图表类型必须匹配数据；避免误导性轴、截断和双轴误用。
- 结论边界：区分相关性和因果，标注不确定性、样本限制和下一步验证。

## 5. Hard Constraints

- 不得跳过数据质量检查。
- 不得编造数据、字段、样本量、统计结果或图表结论。
- 不得隐藏清洗、过滤和口径变化。
- 不得把相关性写成因果。
- 不得输出可识别个人数据，除非用户明确授权且必要。
- 不得在数据不足时给强结论。

## 6. Output Format

最终 prompt 应要求目标模型输出：

```text
数据来源和范围：
数据字典：
数据质量检查：
清洗和过滤规则：
指标定义：
分析方法：
结果表：
图表建议 / 图表：
结论：
限制和不确定性：
下一步验证：
```

## 7. Quality Checklist

- [ ] 是否明确数据来源、字段和时间范围？
- [ ] 是否要求数据质量检查？
- [ ] 是否定义指标口径？
- [ ] 是否记录清洗和过滤规则？
- [ ] 是否要求可复现代码、查询或步骤？
- [ ] 是否要求图表与数据类型匹配？
- [ ] 是否禁止相关性冒充因果？
- [ ] 是否说明限制、不确定性和下一步？

## 8. Common Mistakes

| Common Mistake | Risk | Repair |
| --- | --- | --- |
| 先下结论再找数据 | 分析偏见 | 先数据质量、口径和方法 |
| 字段含义不清就计算 | 指标错误 | 建立数据字典和待确认项 |
| 隐藏清洗规则 | 结果不可复查 | 输出清洗和过滤规则 |
| 相关性写成因果 | 误导决策 | 明确因果边界和验证需求 |

## 9. Reusable Template

```text
请分析 {{data_source}}。

分析问题：
{{analysis_questions}}

字段说明：
{{field_definitions}}

指标口径：
{{metric_definitions}}

时间范围：
{{time_range}}

交付物：
{{deliverable}}

执行要求：
1. 先读取 schema、样例行、字段类型、时间范围和样本量。
2. 做数据质量检查：缺失、重复、异常、类型、极值、口径冲突。
3. 建立数据字典，字段含义未知时标注 [待补充]。
4. 明确清洗、过滤、聚合和指标定义。
5. 用可复现代码、查询或步骤完成分析。
6. 输出图表建议或图表，说明选择原因。
7. 结论必须由数据支持，禁止把相关性写成因果。

输出格式：
数据来源、数据字典、数据质量、清洗规则、指标定义、方法、结果、图表、结论、限制、下一步。
```

## 10. Example

用户原始需求：

```text
分析用户行为 CSV，找出留存下降原因，并输出图表报告。
```

路由判断：
- 主分支：`data-analytics/data-analysis`
- 辅助分支：`data-analytics/visualization-dashboard`, `documents-research/report-writing`
- 风险等级：Medium

高质量 prompt：

```text
请分析 `[data_source]` 的用户行为数据，目标是找出留存下降相关因素。先读取 schema、样例行、字段类型、时间范围和样本量；建立数据字典；检查缺失、重复、异常、时间戳错误和用户 ID 一致性；定义 D1/D7 留存、活跃、转化等指标口径；用可复现代码按渠道、版本、地域和首日行为分组分析。输出数据质量报告、指标表、关键图表、结论、限制和下一步验证。禁止把相关性写成因果；字段含义未知时标注 `[待补充]`。
```

质量检查结果：
- [x] 包含数据质量和字段字典。
- [x] 指标口径可验证。
- [x] 有图表和报告输出。
- [x] 有因果边界。
