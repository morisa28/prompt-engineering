# Finance Investment Analysis

## 1. Purpose

用于生成金融、投资、预算或财务信息分析框架，帮助整理数据、比较风险、解释概念、做情景分析和形成待核验问题。

不用于个性化投资建议、保证收益、直接建议买卖具体资产、预测确定收益或替代持牌金融顾问。

## 2. Trigger Conditions

明确命中：
- 用户要求股票、基金、债券、加密资产、财务规划、预算或投资研究。
- 用户要求比较标的风险、解释财务指标、整理投资报告。
- 用户要求“要不要买/卖”，应改写为信息性分析和风险框架。

可能命中：
- 数据分析涉及收益、价格、估值、现金流或财务指标。
- 报告写作涉及金融材料。

不应命中：
- 用户只要通用数据分析且无金融决策，使用 `data-analysis`。
- 用户要求内幕信息、操纵市场或规避监管，应拒绝相关部分。

相似分支区别：
- `data-analysis` 处理数据方法，本分支控制金融建议边界。
- `report-writing` 可辅助输出投资研究报告格式。

## 3. Required Inputs

必需输入：
- `{{asset_or_topic}}` 标的或主题。
- `{{analysis_goal}}` 分析目标。
- `{{time_horizon}}` 时间范围。
- `{{data_sources}}` 数据来源。
- `{{risk_profile}}` 风险偏好或适用人群边界。
- `{{current_date}}` 数据日期。

可选输入：
- `{{portfolio_context}}` 组合背景。
- `{{assumptions}}` 分析假设。
- `{{metrics}}` 指标。
- `{{not_financial_advice_notice}}` 非投资建议声明。

缺失时处理：
- 最新数据缺失：必须要求查证日期和来源，不能用过期数据冒充当前。
- 风险偏好未知：输出多情景和风险，不给单一路径。
- 数据来源缺失：只能输出分析框架，不能下具体结论。
- 用户要求买卖指令：限制为风险、情景、待核验问题和咨询专业人士。

## 4. Prompt Construction Rules

- 安全声明：明确不构成个性化投资建议，不保证收益。
- 数据来源：标注数据来源、日期和是否需要实时查证。
- 事实/假设/观点：必须分开。
- 指标分析：说明指标定义、计算口径和限制。
- 情景分析：至少包含乐观、中性、悲观或上行/下行风险。
- 风险披露：波动性、流动性、集中度、监管、汇率、利率、杠杆等按场景覆盖。
- 输出边界：不得给确定买卖指令，可给分析框架、关注点、风险提示和专业咨询建议。

## 5. Hard Constraints

- 不得构成个性化投资建议。
- 不得保证收益或承诺风险可控。
- 不得建议用户买入、卖出、持有具体资产作为确定行动。
- 不得使用过期数据不标日期。
- 不得编造财务数据、价格、新闻、评级或监管信息。
- 不得忽略下行风险、费用、税务或流动性限制。

## 6. Output Format

最终 prompt 应要求目标模型输出：

```text
非投资建议声明：
数据来源和日期：
事实：
假设：
分析指标：
情景分析：
主要风险：
需要核验的数据：
适合咨询专业人士的问题：
```

## 7. Quality Checklist

- [ ] 是否明确非个性化投资建议？
- [ ] 是否禁止买卖指令和收益保证？
- [ ] 是否标注数据来源和日期？
- [ ] 是否区分事实、假设、观点和风险？
- [ ] 是否包含多情景分析和下行风险？
- [ ] 是否说明数据缺口和待核验项？
- [ ] 是否避免过度确定结论？
- [ ] 是否建议必要时咨询持牌专业人士？

## 8. Common Mistakes

| Common Mistake | Risk | Repair |
| --- | --- | --- |
| 直接说买/卖 | 合规和用户损失风险 | 改为信息性分析、情景和风险 |
| 不标数据日期 | 使用过期信息误导 | 每个关键数据标来源和日期 |
| 只看上行空间 | 风险失衡 | 输出下行、流动性、监管和情景分析 |
| 混合事实和观点 | 难以复核 | 分栏输出事实、假设、观点 |

## 9. Reusable Template

```text
请对以下金融/投资主题做信息性分析。这不是个性化投资建议，不能保证收益，不能给确定买入/卖出/持有指令。

标的或主题：{{asset_or_topic}}
分析目标：{{analysis_goal}}
时间范围：{{time_horizon}}
数据来源：{{data_sources}}
当前日期：{{current_date}}
风险偏好/适用边界：{{risk_profile}}
分析假设：{{assumptions}}

执行要求：
1. 标注每个关键数据的来源和日期；数据缺失时列为待核验。
2. 区分事实、假设、观点和风险。
3. 分析业务/财务/估值/市场/费用/税务/流动性等相关维度。
4. 输出乐观、中性、悲观或上行/下行情景。
5. 不给确定买卖指令，不承诺收益。

输出格式：
非投资建议声明、数据来源和日期、事实、假设、指标、情景分析、主要风险、待核验数据、咨询专业人士的问题。
```

## 10. Example

用户原始需求：

```text
帮我写 prompt 分析某只股票要不要买。
```

路由判断：
- 主分支：`domain-specific/finance-investment-analysis`
- 辅助分支：`data-analytics/data-analysis`, `documents-research/report-writing`
- 风险等级：High

高质量 prompt：

```text
请对 `[ticker]` 做信息性投资研究。这不是个性化投资建议，不能保证收益，也不能给确定买入/卖出/持有指令。请使用最新可用数据并标注来源和日期；区分事实、假设、观点和风险；分析业务、财务指标、估值、行业、催化因素、费用、流动性、监管和下行风险；输出乐观/中性/悲观情景。若缺少当前价格、财报或新闻来源，列为待核验数据。最后给出适合咨询持牌专业人士的问题。
```

质量检查结果：
- [x] 非投资建议边界明确。
- [x] 禁止买卖指令和收益保证。
- [x] 要求数据来源和日期。
- [x] 区分事实、假设和风险。
