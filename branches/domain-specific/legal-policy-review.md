# Legal Policy Review

## 1. Purpose

用于对合同、政策、条款、隐私政策或合规材料做信息性摘要、义务提取、风险识别和律师咨询问题准备。

不用于替代律师、给最终法律结论、判断一定违法/有效/可签、声称适用于所有司法辖区或生成规避法律责任的建议。

## 2. Trigger Conditions

明确命中：
- 用户要求分析合同风险、政策条款、合规义务、隐私条款。
- 用户要求总结合同、找风险点、准备律师咨询问题。
- 用户要求比较条款义务、期限、违约责任或争议解决。

可能命中：
- 用户要求“看看这个能不能签”，应限制为风险信息和律师问题。
- 用户要求政策审查，可与隐私/数据安全规则组合。

不应命中：
- 用户要求最终法律意见或规避法律执行。
- 用户没有提供文本却要求判断条款风险，应标为阻塞。

相似分支区别：
- `documentation-analysis` 可辅助提取文档结构，但法律边界由本分支控制。
- `report-writing` 可辅助输出正式报告。

## 3. Required Inputs

必需输入：
- `{{jurisdiction}}` 司法辖区或适用法律。
- `{{document_type}}` 文件类型。
- `{{document_text}}` 合同或政策文本。
- `{{transaction_context}}` 合同背景、交易关系或业务场景。
- `{{risk_focus}}` 风险关注点。

可选输入：
- `{{party_role}}` 用户在合同中的角色。
- `{{negotiation_goals}}` 谈判目标。
- `{{not_legal_advice_notice}}` 非法律建议声明。

缺失时处理：
- 司法辖区缺失：必须标注 `[待补充: jurisdiction]`，不得给适用法律结论。
- 合同文本缺失：阻塞，不能分析不存在的条款。
- 合同背景缺失：只做条款信息摘要，不做风险优先级确定结论。
- 隐私或个人数据条款：加入数据最小化、用途、保留、共享和用户权利检查。

## 4. Prompt Construction Rules

- 安全声明：明确不是法律意见，不替代律师。
- 范围定义：按司法辖区、文件类型、用户角色和交易背景限定。
- 条款抽取：提取付款、交付、保密、知识产权、数据处理、违约、终止、责任限制、争议解决等。
- 风险识别：输出 High/Medium/Low 风险、依据、可能影响和需律师确认事项。
- 义务清单：列责任方、行为、期限、触发条件、违约后果。
- 谈判问题：给出可问律师或对方的问题，不给最终法律判断。
- 不确定性：材料不足和辖区不明必须单列。

## 5. Hard Constraints

- 不得替代律师或给最终法律结论。
- 不得声称条款一定有效、无效、违法或可执行。
- 不得忽略司法辖区和交易背景。
- 不得编造法律条文、判例、监管要求或合同文本。
- 不得建议规避法律义务、隐瞒事实或欺骗对方。
- 不得把风险识别写成确定法律意见。

## 6. Output Format

最终 prompt 应要求目标模型输出：

```text
非法律建议声明：
司法辖区和文件背景：
文件概览：
关键条款摘要：
义务 / 权利 / 期限表：
风险清单：
谈判点：
建议咨询律师的问题：
待补充信息：
```

## 7. Quality Checklist

- [ ] 是否明确非法律建议和律师复核边界？
- [ ] 是否要求司法辖区、文件类型和交易背景？
- [ ] 是否基于提供文本而非编造条款？
- [ ] 是否提取义务、权利、期限和后果？
- [ ] 风险等级是否有条款依据？
- [ ] 是否避免最终法律结论？
- [ ] 是否输出律师咨询问题？
- [ ] 隐私/数据条款是否覆盖用途、共享、保留和权利？

## 8. Common Mistakes

| Common Mistake | Risk | Repair |
| --- | --- | --- |
| 直接说“能签/不能签” | 替代法律意见 | 改为风险点、影响和律师问题 |
| 不问司法辖区 | 结论失准 | 辖区作为必填或待补充 |
| 只摘要不抽义务 | 用户无法行动 | 输出义务/期限/后果表 |
| 编造法条 | 严重误导 | 没有来源时写未提供依据 |

## 9. Reusable Template

```text
请对以下法律/政策文件做信息性审阅。这不是法律意见，不能替代律师，不能给最终法律结论。

司法辖区：{{jurisdiction}}
文件类型：{{document_type}}
用户角色：{{party_role}}
交易/业务背景：{{transaction_context}}
风险关注：{{risk_focus}}
文件文本：{{document_text}}

执行要求：
1. 只基于提供文本提取条款，不编造条款或法律依据。
2. 提取关键义务、权利、期限、违约后果和争议解决。
3. 按 High/Medium/Low 标注风险、依据、可能影响和需律师确认事项。
4. 输出谈判点和建议咨询律师的问题。
5. 司法辖区或背景缺失时标注 [待补充]，不得给确定法律结论。

输出格式：
非法律建议声明、司法辖区和背景、文件概览、关键条款、义务表、风险清单、谈判点、律师问题、待补充信息。
```

## 10. Example

用户原始需求：

```text
总结这份服务合同有哪些坑，告诉我该问律师什么。
```

路由判断：
- 主分支：`domain-specific/legal-policy-review`
- 辅助分支：`documents-research/documentation-analysis`, `documents-research/report-writing`
- 风险等级：High

高质量 prompt：

```text
请审阅 `[contract_text]`，文件类型：服务合同，司法辖区：`[待补充: jurisdiction]`，用户角色：`[party_role]`。这不是法律意见，不能替代律师，不能给最终法律结论。只基于合同文本提取付款、交付、验收、保密、知识产权、数据处理、违约、终止、责任限制和争议解决条款；按 High/Medium/Low 标注风险、条款依据、可能影响和需律师确认事项。输出义务/期限表、风险清单、谈判点和建议咨询律师的问题。司法辖区缺失时不得判断条款最终效力。
```

质量检查结果：
- [x] 有非法律建议边界。
- [x] 要求辖区和合同背景。
- [x] 基于文本提取条款。
- [x] 输出律师问题而非最终结论。
