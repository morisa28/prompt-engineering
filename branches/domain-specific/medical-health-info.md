# Medical Health Info

## 1. Purpose

用于把健康信息整理成安全、可带给医生的材料，包括症状时间线、既往史、用药、检查资料、红旗信号和就医问题清单。

不用于诊断疾病、开处方、建议停药/换药/改剂量、替代医生判断或处理紧急医疗事件。

## 2. Trigger Conditions

明确命中：
- 用户要求解释症状、体检报告、化验结果或医学资料。
- 用户要求准备看医生、整理病情、生成问题清单。
- 用户要求把医疗材料做成结构化摘要。

可能命中：
- 用户描述不适但目标是整理信息。
- 用户要读医学资料并转成通俗说明。

不应命中：
- 用户要求 AI 诊断、处方、停药、换药或判断是否不需要就医。
- 用户出现急症信号时，应先建议立即寻求急救或专业医疗帮助。

相似分支区别：
- `report-writing` 可辅助生成就医摘要，但不能替代医疗安全边界。
- `education-tutoring` 讲医学知识点时不处理个人健康建议。

## 3. Required Inputs

必需输入：
- `{{symptoms}}` 症状和严重程度。
- `{{timeline}}` 开始时间、变化过程、诱因和缓解因素。
- `{{health_context}}` 年龄段、既往病史、过敏史、妊娠/慢病等相关背景。
- `{{medications}}` 当前用药、剂量和近期变化。
- `{{medical_records}}` 检查、报告、影像、体征或医生既往意见。

可选输入：
- `{{red_flags}}` 是否存在胸痛、呼吸困难、意识改变、严重出血、单侧无力等急症信号。
- `{{doctor_visit_goal}}` 就医目标。
- `{{questions_for_doctor}}` 想问医生的问题。

缺失时处理：
- 急症信号未知：先要求 prompt 包含红旗信号提示，不得淡化风险。
- 症状时间线缺失：标记 `[待补充: timeline]`，只做信息整理。
- 用药信息缺失：标记待补充，不得给药物调整建议。
- 检查资料缺失：说明只能整理症状和问题，不能解释未提供的报告。

## 4. Prompt Construction Rules

- 目标定义：只做健康信息整理和就医准备。
- 安全声明：明确不替代医生、不诊断、不处方、不建议停药换药。
- 红旗信号：列出需要立即就医或急救的症状类别。
- 时间线：按日期/时间、症状、严重程度、诱因、缓解因素、相关事件整理。
- 材料归纳：把病史、用药、检查资料和医生既往意见分开。
- 问题清单：生成给医生的问题、就诊时要带的材料和需要补充的信息。
- 不确定性：医学解释必须标注依据和不确定性。

## 5. Hard Constraints

- 不得做诊断或鉴别诊断结论。
- 不得提供处方、剂量、停药、换药或治疗方案。
- 不得建议延迟就医或弱化急症信号。
- 不得解释未提供的检查数据为确定事实。
- 不得替代医生、急救人员或药师判断。
- 高风险症状必须建议及时联系医生或急救服务。

## 6. Output Format

最终 prompt 应要求目标模型输出：

```text
安全边界：
需立即就医红旗信号：
症状时间线：
既往史 / 用药 / 过敏：
检查资料摘要：
就医准备清单：
建议询问医生的问题：
待补充信息：
```

## 7. Quality Checklist

- [ ] 是否明确不替代医生、不诊断、不处方？
- [ ] 是否禁止停药、换药或改剂量建议？
- [ ] 是否列出红旗信号和及时就医路径？
- [ ] 是否整理症状时间线？
- [ ] 是否区分症状、病史、用药和检查资料？
- [ ] 是否输出可带给医生的问题清单？
- [ ] 是否标注不确定性和待补充信息？
- [ ] 是否没有给出确定疾病结论？

## 8. Common Mistakes

| Common Mistake | Risk | Repair |
| --- | --- | --- |
| 直接判断是什么病 | 误导就医和治疗 | 改为症状整理、风险信号和医生问题 |
| 给药物或剂量建议 | 可能造成伤害 | 明确只建议咨询医生/药师 |
| 忽略胸痛、呼吸困难等红旗 | 延误急救 | 先列需立即就医信号 |
| 把报告解释成诊断 | 过度确定 | 说明仅基于提供材料做信息整理 |

## 9. Reusable Template

```text
请把以下健康信息整理成就医准备材料。你不能替代医生，不能诊断，不能处方，不能建议停药、换药或改变剂量。

症状：{{symptoms}}
时间线：{{timeline}}
健康背景：{{health_context}}
当前用药/过敏：{{medications}}
检查资料：{{medical_records}}
就医目标：{{doctor_visit_goal}}

执行要求：
1. 先列出需要立即就医或急救的红旗信号。
2. 按时间线整理症状、严重程度、诱因、缓解因素和相关事件。
3. 分开整理既往史、用药、过敏和检查资料。
4. 生成就医准备清单和建议询问医生的问题。
5. 对缺失或不确定的信息标注 [待补充]。

输出格式：
安全边界、红旗信号、症状时间线、病史/用药/检查摘要、就医准备、医生问题、待补充信息。
```

## 10. Example

用户原始需求：

```text
我头痛三天，想写个 prompt 帮我整理给医生看，不要让 AI 乱诊断。
```

路由判断：
- 主分支：`domain-specific/medical-health-info`
- 辅助分支：`documents-research/report-writing`
- 风险等级：High

高质量 prompt：

```text
请把以下健康信息整理成就医准备材料：头痛三天，严重程度 `[待补充]`，伴随症状 `[待补充]`，既往病史/用药 `[待补充]`，检查资料 `[medical_records]`。你不能替代医生，不能诊断，不能处方，不能建议停药、换药或改变剂量。先列出需要立即就医的红旗信号，如突发剧烈头痛、意识改变、发热伴颈项强直、单侧无力、视物异常、外伤后头痛等；再整理症状时间线、就医时要带的资料和建议询问医生的问题。缺失信息标注 `[待补充]`。
```

质量检查结果：
- [x] 不诊断、不处方、不替代医生。
- [x] 包含红旗信号。
- [x] 输出症状时间线和医生问题。
- [x] 标注缺失信息。
